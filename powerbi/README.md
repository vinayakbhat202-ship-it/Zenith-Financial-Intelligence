# Zenith Power BI Reporting & DAX Manual

This manual details how to load the forensic exports generated by Zenith into Power BI Desktop, set up the star schema relationships, implement custom DAX measures, and design the analytics tabs.

---

## 1. Load Data
Import the generated CSV files located in `exports/` into Power BI:
1.  Open **Power BI Desktop**.
2.  Click **Get Data** -> **Text/CSV**.
3.  Load:
    *   `dim_accounts.csv`
    *   `dim_entities.csv`
    *   `dim_users.csv`
    *   `fact_journal_entries.csv`
    *   `fact_audit_log.csv`

---

## 2. Model View Relationships (Star Schema)
Navigate to the **Model view** (left sidebar icon) and configure active relationships as follows:

*   **dim_accounts** (1) $\rightarrow$ **fact_journal_entries** (*)
    *   *Keys*: `dim_accounts.account_key` $\rightarrow$ `fact_journal_entries.account_key`
    *   *Cross-filter direction*: Single
*   **dim_entities** (1) $\rightarrow$ **fact_journal_entries** (*)
    *   *Keys*: `dim_entities.entity_key` $\rightarrow$ `fact_journal_entries.entity_key`
    *   *Cross-filter direction*: Single
*   **dim_users** (1) $\rightarrow$ **fact_journal_entries** (*)
    *   *Keys*: `dim_users.user_key` $\rightarrow$ `fact_journal_entries.user_key`
    *   *Cross-filter direction*: Single
*   **fact_journal_entries** (1) $\rightarrow$ **fact_audit_log** (0/1)
    *   *Keys*: `fact_journal_entries.entry_id` $\rightarrow$ `fact_audit_log.entry_id`
    *   *Cross-filter direction*: Both (allows filtering transaction details by AI determination)

## 3. Power BI DAX Measures
Create a new measure table or create these measures under the `fact_journal_entries` table:

### Total Debits
```dax
Total Debits = SUM(fact_journal_entries[debit_amount])
```

### Total Credits
```dax
Total Credits = SUM(fact_journal_entries[credit_amount])
```

### Double Entry Ledger Balance (Checks if Debits = Credits)
```dax
Ledger Discrepancy = [Total Debits] - [Total Credits]
```

### Total Ledger Value
```dax
Total Ledger Value = [Total Debits]
```

### Average Risk Score
```dax
Average Anomaly Risk = AVERAGE(fact_journal_entries[risk_score])
```

### Anomaly Count
```dax
Total Flagged Anomalies = 
CALCULATE(
    DISTINCTCOUNT(fact_journal_entries[entry_id]),
    fact_journal_entries[risk_score] > 70
)
```

### Anomaly Value Ratio (Percentage of cash affected by anomalies)
```dax
Anomaly Value Ratio = 
VAR FlaggedValue = 
    CALCULATE(
        [Total Ledger Value],
        fact_journal_entries[risk_score] > 70
    )
VAR TotalVal = [Total Ledger Value]
RETURN
    DIVIDE(FlaggedValue, TotalVal, 0)
```

### User Count
```dax
Active Bookkeepers = DISTINCTCOUNT(fact_journal_entries[posted_by])
```

## 4. Visual Dashboard Designs & Tabs

### Tab 1: CFO Treasury Compliance Overview
*   **Cards (KPIs)**:
    *   *Total Ledger Value* | *Average Anomaly Risk* | *Total Flagged Anomalies* | *Ledger Discrepancy* (Should display $0.00).
*   **Donut Chart**: 
    *   *Legend*: `fact_audit_log[llm_determination]`
    *   *Values*: Distinct Count of `entry_id`
    *   *Insight*: Shows distribution of AI Audited classifications (COMPLIANT, SUSPICIOUS, AUDIT_REQUIRED).
*   **Clustered Column Chart**:
    *   *X-Axis*: `dim_accounts[account_class]`
    *   *Y-Axis*: `Average Anomaly Risk`
    *   *Insight*: Pinpoints which general ledger account class (e.g. Asset vs. Expenses) is experiencing the highest statistical anomalies.

### Tab 2: Forensic Auditor Investigative Panel
*   **Decomposition Tree Visual**:
    *   *Analyze*: `Average Anomaly Risk`
    *   *Explain By*: `dim_entities[entity_name]` $\rightarrow$ `dim_users[username]` $\rightarrow$ `dim_accounts[account_name]`.
    *   *Insight*: Allows auditors to drill down from subsidiaries to operators to find anomaly hot spots.
*   **Data Table (Flagged Alerts)**:
    *   *Columns*: `fact_journal_entries[entry_id]`, `fact_journal_entries[transaction_date]`, `fact_journal_entries[posted_by]`, `fact_journal_entries[approved_by]`, `fact_journal_entries[risk_score]`, `fact_audit_log[llm_determination]`, `fact_audit_log[llm_audit_summary]`.
    *   *Filters*: Filter table where `risk_score > 70`.
    *   *Insight*: Provides forensic investigators with the direct context and plain-English AI explanations (RAG results) to quickly understand transaction risks.
